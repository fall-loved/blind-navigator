
Кроссплатформенное мобильное приложение для автономной навигации внутри зданий (университетов, больниц, торговых центров). Проект разработан специально для людей с нарушениями зрения и использует метод пешеходного счисления пути, полностью исключая зависимость от GPS, интернета или внешних Bluetooth-маячков.

Проект построен на **React Native** с использованием платформы **Expo** и строгой типизацией на **TypeScript**.

Утилита создания карт 
      В комплекте с проектом поставляется собственный легковесный визуальный редактор векторных карт mapper.html. Инструмент работает локально прямо в браузере и предназначен для перевода графических планов зданий в строгий JSON-формат, понятный навигационному ядру мобильного приложения. Карты рисуются вручную.
      
Возможности редактора:
            Привязка к плану: Загрузка изображения поэтажного плана в качестве подложки.
            Калибровка пространства: Установка точки отсчета (0,0) и настройка масштаба (метров в 1 пикселе).
            Векторная разметка графа: Расстановка целевых точек (красные узлы) и транзитных путевых точек (синие узлы).
            Отрисовка физических барьеров: Создание непреодолимых стен (отрезков), которые алгоритм лучевого трассирования (mapEngine.ts) будет использовать для предотвращения прокладки маршрутов сквозь препятствия. Поддержка рисования строго под 90° (клавиша Shift).
            Импорт/Экспорт: Мгновенное скачивание готовой топологии в формате .json или загрузка старой карты для продолжения редактирования.
            Как использовать: Просто откройте файл mapper.html в любом современном браузере. Установка сервера не требуется.

Установка и локальный запуск
Требования
   Node.js
   Java Development Kit (JDK) версии 17 или выше
   Android Studio (с установленным Android SDK и настроенными переменными среды ANDROID_HOME)

Шаги установки
   Клонируйте репозиторий любым удобным способом
   Установите зависимости:
      npm install
   Сгенерируйте нативный код (Android/iOS):
   Эта команда создаст папки android и ios, подготовив проект к локальной компиляции.
      npx expo prebuild --clean

Сборка для разработки (Dev Build)
Dev Build — это кастомный клиент, который устанавливается на телефон как обычное приложение, но содержит внутри меню разработчика и поддерживает горячую перезагрузку при изменении JS-кода.
Для его сборки нужно перейти в каталог android, генерируемый при выполнении npx expo prebuild --clean
Далее выполняем ./gradlew assembleDebug и в результате получим apk файл, который необходимо установить на тестовое устройство готовый apk будет находится по пути android/app/build/outputs/apk/debug/app-debug.apk. Перенесите его на ваш Android-смартфон и установите. Далее запустите локальный сервер npx expo start --dev-client и подключитесь к серверу с мобильного устройства указав путь exp://<ip сервера>:8081. Критически важно, чтобы оба устройства находились внутри одной сети, иначе подключения не будет. Это самый быстрый и лёгкий вариант для тестирования изменений в приложении

Сборка готового приложения (Release APK)
Когда разработка завершена, вам нужен финальный, оптимизированный .apk файл, который работает полностью автономно (без запущенного на компьютере сервера).Перейдите в директорию android и запустите сборку релиза. В этом режиме код минифицируется, а ресурсы оптимизируются. В дериктории android выполните ./gradlew assembleRelease готовый apk будет находится по пути android/app/build/outputs/apk/release/app-release.apk. Этот файл можно использовать в "полевых" условиях для проверки автономной индор-навигации.
      
⚠️ Важное примечание по аппаратному обеспечению
Ввиду специфики работы метода пешеходного счисления пути внутри железобетонных конструкций, точность навигации напрямую зависит от стабильности магнитометра (компаса) в смартфоне. Перед началом использования или тестирования в новом здании настоятельно рекомендуется калибровать компас смартфона ("рисовать восьмерку" в воздухе).
